## 三维写作分析

### structure (85)

**分析：**
框架类型：定义→分类→技巧→分发，清晰的"是什么-怎么做-如何推广"递进结构。

开头：用误解引出问题（"Skills 只不过是 markdown 文件"），用"我们已经用了几百个"建立权威，3 句话抓住注意力。

中段：九类技能分类用标题+示例形式展开，信息密度高但条理清晰。编写技巧部分采用"问题→解决方案"模式（如"不要说显而易见的事"→"踩坑点章节"→"文件系统"），层层递进。

结尾：谦逊定位（"不是权威指南，而是实用技巧合集"）+ 呼吁行动（"动手开始做、不断试验"），开放且有启发性。

### style (76)

**分析：**
语气：权威但谦逊的技术导师口吻，"我们发现"、"我们总结"增强可信度，"不是终极清单"、"我们都在摸索"降低距离感。

句式特征：长短句结合，标题多为祈使句（"不要说显而易见的事"、"建一个踩坑点章节"），指令性强。大量使用破折号补充说明（如"——它们是文件夹，可以包含脚本、资源文件"），提供额外上下文。

修辞：对比修辞突出价值（"不只是文本文件——它们是文件夹"），类比（"渐进式披露"借用了 UI 设计概念），反复强调"踩坑点"作为核心概念。

### technique (80)

**分析：**

核心写作技巧 1：分类框架法。将抽象的 Skills 使用经验归纳为九个清晰类别，每个类别配 3 个具体示例，降低认知负荷。这种"框架+示例"的组合是技术写作的黄金范式。

核心写作技巧 2：认知反常识制造。多次提出反直觉观点（如"description 字段是给模型看的，不是给人看的"），通过打破读者预期增强记忆点。这是高阶写作技巧。

证据运用：大量真实案例（billing-lib、standup-post、/careful 等），所有建议都来自"几百个内部 Skills"的实践，非理论推演。引用官方文档和外部资源（Skilljar 课程、GitHub gist）增强权威性。

## 可仿写元素

- **九类分类框架**：将复杂工具的用法归纳为 5-10 个清晰类别，每个类别配 3 个真实示例。适用于任何技术工具、方法论或框架的教程。

- **踩坑点章节设计**：在任何教程中设置专门的"踩坑点/gotchas"章节，聚焦"常见失败点"而非"理论正确性"。这比罗列成功经验更有价值。

- **反常识观点植入**：在技术写作中故意提出 1-2 个反直觉观点（如"description 是给 AI 看的"），通过打破认知偏差增强文章记忆点。

- **渐进式披露结构**：先用大框架建立地图，再提供详细参考资料供按需探索。避免一次性倾倒所有信息。

- **内部视角权威性**：开篇即强调"来自内部/大规模实践/真实数据"，建立稀缺性。如"我们从几百个 Skills 中提炼出"比"我们认为"更有说服力。
