## 可仿写元素

- **类比开篇法**：用"最近多看 X 文章，就会反复遇到同一个词：Y"制造共鸣，然后指出"这个词越讲越糊"，最后引出自己的清晰定义。这种写法适合技术概念的澄清文章。

- **三层对比框架**：在讲解相关概念时，用"Prompt 更像 A。Context 更像 B。Harness 更像 C。"的平行句式，配合具体比喻，让读者快速建立认知梯度。

- **六层拆解结构**：对于复杂系统，可以拆解为 6-8 个独立层次，每个层次都用"问题是什么、为什么重要、成熟系统怎么做"的三段式结构展开。这种结构适合技术综述和架构讲解。

- **历史演进收尾**：文章结尾不简单总结，而是将话题放入更长的时间维度（"软件工程过去 30 年"）和更广的行业视角（"复杂性中心迁移"），提升文章的格局。

- **实战原则清单**：在理论分析后，用 5-7 条具体建议作为收尾（"先让一条最短链路跑通"、"工具不是越多越好"等），给读者明确的行动指南。每条建议都用反直觉的洞察吸引注意力（"工具一多，模型的调用质量会明显下降"）。
