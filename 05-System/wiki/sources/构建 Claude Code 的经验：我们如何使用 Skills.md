# 构建 Claude Code 的经验：我们如何使用 Skills

## 核心论点

这是 Anthropic 内部团队分享的 Claude Code 使用经验，基于数百个真实 Skills 的实践总结。文章提供了系统的分类框架和实用的编写技巧。

## Skills 分类体系（9 大类）

1. **项目规范类**：编码标准、架构决策
2. **工具集成类**：外部工具、API 集成
3. **代码生成类**：模板、脚手架
4. **测试相关类**：测试策略、用例生成
5. **文档相关类**：文档生成、维护
6. **部署相关类**：CI/CD、部署流程
7. **调试类**：问题诊断、故障排除
8. **重构类**：代码优化、技术债务
9. **学习辅助类**：知识传递、培训

## 关键洞察

### 1. Description 是给模型看的，不是给人看的

- 不要在 description 中写复杂的说明
- 模型需要的是简洁、明确的指令
- 示例 > 说明

### 2. 踩坑点聚焦

- 列出常见的错误和如何避免
- 提供具体的反例
- 解释为什么这样做是错的

### 3. 渐进式披露

- 从简单到复杂组织信息
- 让模型逐步理解上下文
- 避免信息过载

### 4. 具体案例驱动

- 使用真实的代码示例
- 展示输入和输出
- 提供可运行的片段

## 最佳实践

### 编写 Skills 的原则

1. **明确性**：清楚地告诉模型要做什么
2. **简洁性**：避免冗余和不必要的信息
3. **可测试性**：Skills 应该可以独立测试
4. **可维护性**：定期更新和优化

### 避免

- 过于抽象的描述
- 没有示例的理论说明
- 过时的信息
- 复杂的逻辑嵌套
