# 工具系统

**英文名**: Tool System
**类型**: Agent Harness组件
**比喻**: Agent的手

## 概述

工具系统是Agent与外部世界交互的核心机制。根据[Agent Harness 综述：同一个模型，为什么做出来的 Agent 差这么远](../sources/Agent Harness 综述：同一个模型，为什么做出来的 Agent 差这么远.md)的论述，工具系统不只是"给几个函数名"那么简单，它至少要管理四件事：工具注册、参数校验、执行环境隔离、结果回写。

## 四大核心职责

### 1. 工具注册（Tool Registration）
**功能**: 让模型知道有哪些工具可用

**实现方式**:
```python
tools = {
    "read_file": {
        "description": "读取文件内容",
        "parameters": {
            "path": {"type": "string", "description": "文件路径"}
        }
    },
    "write_file": {
        "description": "写入文件内容",
        "parameters": {
            "path": {"type": "string"},
            "content": {"type": "string"}
        }
    }
}
```

### 2. 参数校验（Parameter Validation）
**功能**: 确保工具调用参数正确

**校验维度**:
- 参数类型检查
- 参数范围验证
- 必需参数检查
- 参数格式验证

**示例**:
```python
def validate_tool_params(tool_name, params):
    # 类型检查
    if not isinstance(params["path"], str):
        raise TypeError("path must be string")
    
    # 范围检查
    if "max_results" in params and params["max_results"] < 0:
        raise ValueError("max_results must be non-negative")
    
    # 必需参数
    required = ["path"]
    for param in required:
        if param not in params:
            raise ValueError(f"Missing required param: {param}")
```

### 3. 执行环境隔离（Execution Isolation）
**功能**: 保护系统安全，防止工具调用影响主进程

**隔离方式**:
- **进程隔离**: 在独立进程中运行工具
- **容器隔离**: 使用Docker等容器技术
- **沙箱**: 受限的执行环境
- **权限控制**: 限制文件系统、网络访问

**示例**:
```python
def execute_tool_sandboxed(tool_name, params):
    # 在受限环境中执行
    with Sandbox(allow_network=False, 
                 allow_filesystem="/tmp"):
        result = tools[tool_name](**params)
    return result
```

### 4. 结果回写（Result Writing）
**功能**: 将工具执行结果转换成模型能理解的Observation

**回写格式**:
```python
observation = {
    "tool": "read_file",
    "status": "success",
    "result": "文件内容...",
    "metadata": {
        "file_size": 1024,
        "modified_time": "2026-04-19"
    }
}
```

## 工具分类

### Claude Code分类（六类）
1. **文件操作**: read, write, edit
2. **搜索**: grep, find
3. **执行**: bash, script
4. **Web访问**: browser, http_request
5. **代码智能**: analyze, refactor
6. **子Agent派生**: spawn_subagent

### OpenAI Agents SDK分类（三类）
1. **函数工具**: 传统的函数调用
2. **托管工具**: 由平台托管的服务
3. **MCP工具**: 基于MCP协议的工具

## 关键洞察

### 模型知道能用什么 ≠ 能用好
> 模型知道可以调用什么工具，只是起点。它能不能在合适的时候调用、带着正确参数调用、在失败后继续恢复，才是Harness的事情。

### 工具数量与质量
**警告**: 
> 当工具数超过10个且职责重叠时，模型的调用质量会明显下降。

**建议**: 只暴露当前步骤会用到的最小工具集

## 最佳实践

### 1. 渐进式暴露工具
```python
# 初始阶段：基础工具
initial_tools = ["read", "write", "search"]

# 需要时：专业工具
if task == "deployment":
    tools.extend(["deploy", "rollback", "monitor"])
```

### 2. 工具描述优化
```python
# 差的描述
"tool": {
    "name": "exec",
    "description": "执行命令"
}

# 好的描述
"tool": {
    "name": "exec",
    "description": "在bash shell中执行命令，返回stdout和stderr",
    "use_cases": ["运行测试", "构建项目", "安装依赖"],
    "risks": ["可能修改文件系统", "可能很慢"]
}
```

### 3. 失败处理
```python
def execute_tool_with_fallback(tool, params, max_retries=3):
    for attempt in range(max_retries):
        try:
            return tools[tool](**params)
        except TemporaryError:
            if attempt == max_retries - 1:
                return {"status": "failed", "error": "..."}
            continue
        except PermanentError:
            return {"status": "failed", "error": "..."}
```

## 常见陷阱

### 1. 工具过多
- 模型选择困难
- 参数理解错误
- 上下文占用过大

### 2. 工具职责不清
- 同一功能多个工具
- 工具边界模糊
- 模型不知道用哪个

### 3. 缺少工具反馈
- 执行失败没有详细信息
- 错误消息对模型不友好
- 无法判断失败原因
