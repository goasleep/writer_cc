# 事实记忆

**事实记忆（Declarative / Factual Memory）** 是 Agent 系统中记录稳定事实信息的长期资产层，回答的是"我是谁、环境是什么、用户偏好是什么"。

## 核心特征

- **内容**：用户偏好、项目路径、常用命令、环境约定、工具 quirks。
- **形式**：通常是小型结构化文件（如 Hermes 的 `MEMORY.md` / `USER.md`）。
- **召回方式**：在会话启动时作为 frozen snapshot 注入 system prompt，全程可见。
- **容量控制**：需要刻意克制，避免污染上下文。Hermes 的默认值仅约 800 / 500 tokens。

## 与相关概念的区别

| 层级 | 记录对象 | 召回时机 | 典型载体 |
|------|---------|---------|---------|
| 事实记忆 | 稳定事实、偏好、环境 | 会话启动时注入 | `MEMORY.md`, `USER.md` |
| 会话检索 | 历史对话、具体事件 | 按需搜索 | SQLite + FTS5, transcript |
| 过程记忆 | 可复用流程、做事方法 | 任务匹配时加载 | skills, SOP |

## 工程要点

- 任务进展、临时 TODO、完成记录**不应**进入事实记忆，应留在历史 transcript 中。
- 写入时应做安全扫描，防止 prompt injection 和凭据泄露进入 system prompt。
- 为保护 prompt cache，新写入的记忆通常等下一轮会话才生效。
