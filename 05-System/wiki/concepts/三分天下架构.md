# 三分天下架构

**提出者**: 老冯
**提出时间**: 2026年
**来源**: [三分天下：为什么Agent Memory框架是死路](../sources/三分天下：为什么Agent Memory框架是死路.md)

## 概述

"三分天下"是对AI Agent终局架构的判断：一个成熟的Agent，在终局状态下，会形成**三层彻底解耦**的架构——模型层、Harness层、数据库层，各自独立，各自为王。

## 架构图

```
┌─────────────────────────────────────────────────┐
│                   Agent应用                      │
├─────────────────────────────────────────────────┤
│  MODEL_URL=https://api.anthropic.com/v1         │  ← 模型层（智力）
│  提供者: Anthropic, OpenAI, DeepSeek等          │
├─────────────────────────────────────────────────┤
│  Harness (Claude Code/Cursor/Devin/...)         │  ← Harness层（驾驭执行）
│  - 加载Skills                                   │
│  - 组织context                                  │
│  - 调用工具                                     │
│  - 处理循环                                     │
├─────────────────────────────────────────────────┤
│  DB_URL=postgres://user:pass@host:5432/memory   │  ← 数据库层（记忆）
│  提供者: PostgreSQL, SQLite等                   │
└─────────────────────────────────────────────────┘
```

## 三层职责

### 模型层 - 提供智力

**代表**: Anthropic Claude, OpenAI GPT, DeepSeek等

**职责**:
- 提供通用智能
- 理解和生成自然语言
- 推理和决策

**特点**:
- 算力短期关键长期摊平
- 模型中期关键长期平权
- 开源模型一年一个台阶
- GPT-5和DeepSeek V4的差距比GPT-4时代小了一大截
- 再过两年，Agent能用的模型大概率是个丰俭由人的菜市场

### Harness层 - 驾驭执行

**代表**: Claude Code, Cursor, Devin, OpenClaw, Hermes, Letta/MemGPT等

**职责**:
- 把模型套起来，驾驭它去完成具体任务
- 加载和管理Skills
- 组织和优化context
- 协调工具调用
- 处理执行循环

**特点**:
- 这块还在摸索和成形
- Claude Code通过开源掀翻了壁垒
- 局势仍在剧烈变动中

### 数据库层 - 管理记忆

**代表**: PostgreSQL, SQLite等

**职责**:
- 存储和管理Agent的记忆
- 提供可靠持久化
- 支持复杂检索

**特点**:
- **不在AI冲击范围内**（物理世界持久化，非信息加工）
- 确定性最高的位置
- PostgreSQL将成为通用Agent记忆层的终局

## 为什么是三分天下

### 动力学

这套格局不是架构师的设计品味，背后有很朴素的动力学：

**真正的壁垒在私有数据**:
- 算力会摊平
- 模型会平权
- 企业不会允许核心数据被单一服务商锁定
- 三方制衡是必然结果

**博弈终点**:
- 模型厂商管智能
- Harness管驾驭执行
- 数据库厂商管记忆
- 三方互不吞并、互相制衡

### 为什么没有Memory框架的位置

Memory框架试图在三者之间插入一层，但：
- 产业结构上没位置（三方互相制衡，不给中间商留空）
- 技术上无壁垒（就是建表和SQL）
- 方法论上撑不住（The Bitter Lesson）

## 终局时间线

**现阶段**（2026年）:
- 本地Agent: SQLite或文件系统完全够用
- 单机轻量Agent: 过度工程才用PG

**演化阶段**:
- 一旦需要跨Agent协作、跨设备持久化、跨组织可迁移
- 也就是需要"通用可迁移的记忆层"
- 局势就会收敛到PostgreSQL

**终局**（两年后）:
- 独立Memory框架赛道可能消失
- 三分天下格局稳定
- PostgreSQL成为通用记忆层事实标准
