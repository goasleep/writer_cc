# 会话检索

**会话检索（Session Search / Episodic Retrieval）** 是 Agent 系统中按需从历史对话和任务记录中召回信息的能力，解决的是"上次我们聊过什么、怎么修的"这类问题。

## 核心特征

- **内容**：完整的历史会话消息、任务过程、错误排查记录。
- **形式**：通常由数据库（如 SQLite）配合全文检索（如 FTS5）实现。
- **召回方式**：用户提及"上次""之前"时，通过工具调用按需搜索，而非常驻上下文。
- **定位**：更像档案室，不是随身备忘录。

## 工程实现要点

- 使用 WAL 模式支持并发读写。
- 通过 trigger 自动维护 FTS 索引。
- 截断策略优先覆盖匹配位置最多的窗口，控制召回长度。
- 宽查询无结果时，可拆词并行搜索以提高召回率。

## 设计原则

- 历史信息不适合每次都塞进 system prompt，否则会浪费 token 并破坏 prompt cache 稳定性。
- 需要时少靠猜，让 Agent 主动搜索历史 transcript。
