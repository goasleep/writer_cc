# 守护进程

**英文名**: Daemon
**发音**: /ˈdiːmən/ 或 /ˈdeɪmən/
**类型**: 后台进程
**运行环境**: Linux/Unix系统

## 概述

守护进程是运行在系统后台的特殊进程，通常在系统启动时自动运行，在系统关闭时退出。根据[不懂 fork 与 写时复制，别再说你懂 Linux 内核了](../sources/不懂 fork 与 写时复制，别再说你懂 Linux 内核了.md)的论述，守护进程的创建流程充分利用了fork和写时复制技术来降低创建开销并保持独立性。

## 特点

### 1. 后台运行
- 不与任何终端关联
- 不受终端操作影响
- 用户登录/退出不影响其运行

### 2. 长期运行
- 系统启动时启动
- 系统关闭时退出
- 可能永久运行

### 3. 无用户交互
- 没有控制台输入
- 通过日志文件记录信息
- 通过信号、配置文件等控制

### 4. 特殊权限
- 通常以root权限运行
- 继承父进程的资源

## 创建步骤

### 标准流程（两次fork）

#### 第一步：第一次fork
```c
pid_t pid = fork();

if (pid < 0) {
    // fork失败
    return -1;
}

if (pid > 0) {
    // 父进程退出
    exit(0);
}

// 子进程继续...
```

**目的**:
- 父进程退出，子进程被init（PID 1）收养
- 子进程成为新会话的首进程
- 脱离原终端控制

**COW的作用**:
- 利用写时复制快速创建子进程
- 共享父进程内存，最小化开销

#### 第二步：创建新会话
```c
// 创建新会话，脱离控制终端
setsid();
```

**效果**:
- 成为新会话的首进程
- 成为新进程组的首进程
- 脱离原控制终端

#### 第三步：第二次fork
```c
pid = fork();

if (pid > 0) {
    // 第一个子进程退出
    exit(0);
}

// 第二个子进程继续...
```

**目的**:
- 确保进程不是会话首进程
- 防止再次申请控制终端
- 增强独立性

**COW的作用**:
- 再次利用写时复制
- 进一步降低资源占用

#### 第四步：其他设置
```c
// 1. 改变工作目录
chdir("/");

// 2. 重置文件权限掩码
umask(0);

// 3. 关闭所有文件描述符
for (int i = 0; i < MAXFD; i++) {
    close(i);
}

// 4. 重定向标准输入/输出/错误
open("/dev/null", O_RDWR);  // stdin
dup(0);                     // stdout
dup(0);                     // stderr
```

## 完整示例

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/stat.h>
#include <sys/types.h>

int create_daemon() {
    // 第一次fork
    pid_t pid = fork();
    if (pid < 0) {
        perror("fork");
        return -1;
    }
    
    if (pid > 0) {
        // 父进程退出
        exit(0);
    }
    
    // 子进程创建新会话
    if (setsid() < 0) {
        perror("setsid");
        return -1;
    }
    
    // 第二次fork
    pid = fork();
    if (pid < 0) {
        perror("fork");
        return -1;
    }
    
    if (pid > 0) {
        // 第一个子进程退出
        exit(0);
    }
    
    // 第二个子进程（真正的守护进程）
    
    // 改变工作目录
    if (chdir("/") < 0) {
        perror("chdir");
        return -1;
    }
    
    // 重置文件权限掩码
    umask(0);
    
    // 关闭所有文件描述符
    for (int i = 0; i < sysconf(_SC_OPEN_MAX); i++) {
        close(i);
    }
    
    // 重定向标准输入/输出/错误
    open("/dev/null", O_RDWR);
    dup(0);
    dup(0);
    
    return 0;
}

int main() {
    if (create_daemon() < 0) {
        fprintf(stderr, "Failed to create daemon\n");
        return 1;
    }
    
    // 守护进程的主循环
    while (1) {
        // 执行守护任务
        sleep(1);
    }
    
    return 0;
}
```

## 常见守护进程

### 系统守护进程
- **init**: PID 1，所有进程的祖先
- **systemd**: 现代Linux的init系统
- **sshd**: SSH服务器
- **cron**: 定时任务调度
- **rsyslog**: 日志系统

### 网络服务
- **httpd/nginx**: Web服务器
- **mysqld**: MySQL数据库
- **ntpd**: 时间同步

## 日志管理

### 日志文件位置
- **系统日志**: `/var/log/syslog`, `/var/log/messages`
- **应用日志**: `/var/log/<app>/`
- **错误日志**: `/var/log/errlog`

### 日志管理方式
```c
// 示例：写入日志文件
FILE *log_file = fopen("/var/log/mydaemon.log", "a");
if (log_file != NULL) {
    fprintf(log_file, "Daemon started at %s", ctime(&now));
    fclose(log_file);
}

// 使用syslog
#include <syslog.h>
openlog("mydaemon", LOG_PID, LOG_DAEMON);
syslog(LOG_INFO, "Daemon started");
closelog();
```

## 信号处理

### 常用信号
```c
#include <signal.h>

void signal_handler(int sig) {
    switch (sig) {
        case SIGHUP:  // 重新加载配置
            reload_config();
            break;
        case SIGTERM:  // 优雅终止
            cleanup();
            exit(0);
        case SIGINT:   // 同SIGTERM
            cleanup();
            exit(0);
    }
}

// 注册信号处理
signal(SIGHUP, signal_handler);
signal(SIGTERM, signal_handler);
signal(SIGINT, signal_handler);
```

## 性能与资源

### COW的优势
在守护进程创建中，写时复制技术带来以下优势：

1. **快速启动**: 两次fork都能快速完成
2. **低内存占用**: 初始共享内存，最小化资源使用
3. **高稳定性**: 减少内存分配失败的风险

### 资源特性
- **低CPU使用**: 大部分时间处于睡眠状态
- **稳定内存**: 不频繁分配/释放内存
- **长连接**: 保持网络连接打开

## 管理工具

```bash

systemctl status <service>
```

### 控制守护进程
```bash
# systemd系统
systemctl start <service>
systemctl stop <service>
systemctl restart <service>
systemctl reload <service>

# 传统init系统
service <service> start
service <service> stop
service <service> restart
```

### 配置文件
- **systemd**: `/etc/systemd/system/*.service`
- **传统init**: `/etc/init.d/*`

## 常见问题

### 1. 守护进程无法启动
**检查**:
- 配置文件语法
- 日志文件权限
- 端口占用
- 依赖服务

### 2. 守护进程意外退出
**诊断**:
- 查看系统日志
- 检查core dump
- 验证信号处理

### 3. 资源泄漏
**监控**:
```bash
ps aux | grep <daemon>
top -p <daemon_pid>
lsof -p <daemon_pid>
```
