# 容器隔离

容器隔离是一种通过容器技术（如 Docker）将应用程序及其依赖项与宿主系统隔离的安全措施。

## 在 AI Agent 中的应用

### [[Hermes Agent]] 的安全机制

Hermes Agent 使用容器隔离来增强 AI 助理的安全性：

1. **执行隔离**：AI 执行的命令在容器中运行，不直接影响宿主系统
2. **文件系统隔离**：限制 AI 能访问的目录，保护敏感数据
3. **网络隔离**：控制 AI 的网络访问权限

## 安全优势

### 防止意外破坏
- AI 被诱导执行危险命令时，容器提供额外保护层
- 即使 AI "犯浑"，也不至于清空宿主系统文件

### 防止凭证泄露
- 容器内无法访问宿主的敏感配置
- API Key 等凭证不会被意外暴露

### 受保护目录
- 可指定某些目录禁止访问
- 防止路径穿越攻击

## 与传统 AI Agent 的对比

| 安全措施 | [[OpenClaw]] | [[Hermes Agent]] |
|---------|-------------|------------------|
| 命令执行 | 模型判断 | 框架级审批 + 容器隔离 |
| 文件访问 | 依赖模型 | 受保护目录 + 路径穿越防护 |
| 凭证保护 | 无专门机制 | 凭证泄露拦截 |

## 技术实现

- 容器技术：Docker、Podman
- 隔离级别：进程级、网络级、文件系统级
- 权限管理：用户命名空间、cgroups
