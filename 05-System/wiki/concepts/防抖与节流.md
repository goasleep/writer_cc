# 防抖与节流

## 防抖（Debounce）

等待事件停止触发后的一定延迟时间，才执行目标函数。常用于搜索框输入场景，避免用户每输入一个字符就发起一次 API 请求。

## 节流（Throttle）

在指定时间间隔内，最多只执行一次目标函数。常用于滚动加载、窗口 resize 等持续触发的事件。

## 价值

- 显著减少无效 API 请求
- 降低后端压力
- 提升用户体验与应用响应速度
