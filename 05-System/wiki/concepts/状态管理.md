# 状态管理

**英文名**: State Management
**类型**: Agent Harness组件
**所属层**: Harness六大承重层之一

## 概述

状态管理负责跟踪任务的执行进度和中间结果。根据[Agent Harness 综述：同一个模型，为什么做出来的 Agent 差这么远](../sources/Agent Harness 综述：同一个模型，为什么做做出的 Agent 差这么远.md)的论述，任务一旦变长，状态管理就从"有最好"变成"没有不行"。

## 核心问题

系统需要知道：
1. **当前做到哪一步**
2. **失败后从哪恢复**
3. **哪些中间产物值得保留**
4. **哪些只是临时噪音**

## 为什么重要

### 短任务 vs 长任务
| 任务类型 | 状态管理需求 | 失败代价 |
|---------|-------------|---------|
| 短任务（1-3步） | 可选 | 低，重新执行即可 |
| 中等任务（4-10步） | 推荐 | 中等，部分重复 |
| 长任务（10+步） | **必需** | 高，从头开始代价大 |

### 误差累积
> 10步流程，如果每步成功率都是99%，最终全链路成功率只有约90%。任务再拉长一点，误差就会开始明显堆积。

## 状态类型

### 1. 执行状态（Execution State）
```python
execution_state = {
    "current_step": 5,
    "total_steps": 10,
    "status": "in_progress",
    "last_action": "write_test",
    "next_action": "run_test"
}
```

### 2. 中间结果（Intermediate Results）
```python
intermediate_results = {
    "files_created": ["test1.py", "test2.py"],
    "tests_passed": 8,
    "tests_failed": 2,
    "bugs_fixed": 3
}
```

### 3. 检查点（Checkpoints）
```python
checkpoints = {
    "step_3": {
        "timestamp": "2026-04-19T10:30:00",
        "state": execution_state.copy(),
        "files_snapshot": [...],
        "git_commit": "abc123"
    }
}
```

## 实现方式

### 1. 结构化状态对象
```python
class TaskState:
    def __init__(self):
        self.current_step = 0
        self.steps_completed = []
        self.steps_failed = []
        self.artifacts = {}
        self.checkpoints = []
    
    def save_checkpoint(self, step):
        checkpoint = {
            "step": step,
            "state": self.__dict__.copy(),
            "timestamp": datetime.now()
        }
        self.checkpoints.append(checkpoint)
    
    def restore_from_checkpoint(self, checkpoint_id):
        checkpoint = self.checkpoints[checkpoint_id]
        self.__dict__.update(checkpoint["state"])
```

### 2. Git Commit作为检查点
```bash
# 每个关键步骤后提交
git commit -m "Checkpoint: step 3 completed"
git tag "checkpoint-step-3"

# 恢复时
git reset --hard checkpoint-step-3
```

### 3. 进度文件
```python
# 保存进度
with open("progress.json", "w") as f:
    json.dump({
        "current_step": 5,
        "completed_steps": [1,2,3,4],
        "pending_steps": [6,7,8,9,10],
        "artifacts": {...}
    }, f)

# 恢复进度
with open("progress.json", "r") as f:
    progress = json.load(f)
```

## 恢复策略

### 1. 从失败点恢复
```python
# 检测失败步骤
failed_step = find_failed_step(checkpoint)

# 回退到上一个检查点
restore_from_checkpoint(failed_step - 1)

# 修改策略后重试
retry_with_different_strategy(failed_step)
```

### 2. 跳过非关键步骤
```python
# 如果某些步骤失败不影响整体，可以跳过
if step.is_optional and step.failed:
    mark_step_skipped(step)
    continue_to_next_step()
```

### 3. 部分重做
```python
# 只重做失败的步骤，保留成功的结果
for step in steps:
    if step.status == "failed":
        retry_step(step)
    else:
        keep_step_result(step)
```

## 设计原则

### 1. 幂等性（Idempotence）
```python
# 同一步骤多次执行结果一致
def execute_step(step_id):
    if step_already_completed(step_id):
        return get_cached_result(step_id)
    return execute_and_save(step_id)
```

### 2. 可恢复性（Recoverability）
```python
# 任何步骤失败后都能恢复
def execute_with_recovery(step):
    try:
        return execute_step(step)
    except Exception as e:
        save_checkpoint_before_failure(step, e)
        return recover_from_checkpoint(step)
```

### 3. 可观察性（Observability）
```python
# 清晰记录每一步的状态
def log_step(step, status, result):
    logger.info({
        "step": step.id,
        "status": status,
        "result": result,
        "timestamp": now()
    })
```

### 1. 检查点策略
```python
# 在关键点设置检查点
checkpoint_triggers = [
    "每个主要阶段完成后",
    "高风险操作前",
    "长时间运行的任务中",
    "状态变更时"
]
```

### 2. 状态压缩
```python
# 对旧状态进行压缩
def compress_old_states(states):
    # 保留最近的详细状态
    recent_states = states[-10:]
    
    # 压缩旧状态为摘要
    old_summary = summarize_states(states[:-10])
    
    return recent_states + [old_summary]
```

### 3. 噪音过滤
```python
# 区分有价值信息和噪音
def is_noise(event):
    return (
        event.is_debug_message or
        event.is_transient_error or
        event.is_duplicate_info
    )
```
