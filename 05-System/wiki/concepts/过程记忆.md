# 过程记忆

**过程记忆（Procedural Memory / Skill Memory）** 是 Agent 系统中记录"这类事情下次该怎么做"的长期资产层。与事实记忆和会话检索不同，它记录的是可复用的做事方法和流程，而非静态事实或历史事件。

## 核心特征

- **内容**：复杂任务的工作流、排查顺序、检查清单、最佳实践。
- **形式**：以 skill 为载体，通常包含 YAML frontmatter 和 Markdown 正文。
- **召回方式**：任务匹配时由 Agent 主动调用 `skill_manage` 创建、检索、修补或删除。
- **生命周期**：需要持续维护，过时的 skill 会误导 Agent。

## 与事实记忆的区别

| | 事实记忆 | 过程记忆 |
|--|---------|---------|
| 回答的问题 | 环境是什么、用户偏好是什么 | 这类事该怎么做 |
| 典型内容 | `MEMORY.md`, `USER.md` | skills, SOP |
| 更新频率 | 较低，偏好和约定相对稳定 | 较高，需要随经验迭代 |
| 可执行性 | 声明式信息 | 可执行流程和指导 |

## 工程实现要点

- 创建阈值：复杂任务（如 5 次以上工具调用）、棘手错误、非平凡 workflow。
- 安全扫描：每次 create / edit / patch 后做 `_security_scan_skill()`，失败则回滚。
- 后台 review：由独立 review agent 在任务结束后判断是否有可沉淀经验。
- 权限边界：外部 skills 可只读扫描，Agent 创建的 skill 写回主目录。

## 设计原则

- 简单一次性任务无需保存为 skill。
- skill 过时、错误或不完整时，应立即 patch 或删除。
- 错误经验也可能被固化，因此需要验证步骤和失败模式的记录。
